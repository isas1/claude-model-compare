Model Compare — Usage

Session-level usage stats mined from local Claude Code transcripts. What each model gets used for and how much, not which one is "better."

Read this before drawing conclusions: this page is descriptive, not causal. Differences between models mostly reflect routing, task mix, and personal habits — which model got picked for which kind of work — not model capability or quality. Nothing here ranks or scores the models against each other.

Overview

Session count, median output tokens, median active duration, median user turns per model. Toggle chips to add models beyond the three defaults.

Comparison table

Models as columns, metrics as rows. Median is primary (bold); mean shown underneath in muted text.

Multi-model caveat: a session that used more than one model produces one row per model in the underlying data — so a single session with N models appears in N model columns here (once per model it touched), not just once.

Each point is one (session, model) row. Hover for session id, project, and output tokens. Output tokens only — cache tokens are never included here.
