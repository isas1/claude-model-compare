Model Compare — Usage

Session-level usage stats mined from local Claude Code transcripts. What each model gets used for and how much, not which one is "better."

Read this before drawing conclusions: this page is descriptive, not causal. Differences between models mostly reflect routing, task mix, and personal habits — which model got picked for which kind of work — not model capability or quality. Nothing here ranks or scores the models against each other.

Suggested use — observed patterns

How to read this: these are observations about YOUR sessions, not claims about the models. Which model ran a session was decided by routing and habit, not a fair test — so this reflects what you reached for, not what any model is best at. Small cohorts (some models have well under 50 sessions) make these patterns noisy. This panel never ranks models or recommends one over another.

Overview

Session count, median output tokens, median active duration, median user turns per model. Toggle chips to add models beyond the three defaults. All sessions — the filter bar below applies to the comparison table only.

Interaction shape

Per-row ratios (autonomy/verbosity proxies), medianed across each model's cohort — not pooled, so one large session can't dominate. Rows with a zero denominator are excluded from that ratio; excluded counts shown.

Tool-mix fingerprint

Each model's tool calls (main-chain only) bucketed into explore / edit / execute / orchestrate / other, normalized to % of that model's total calls. This is a fingerprint of how a model tends to work, not a quality signal.

Comparison table

Models as columns, metrics as rows. Median is primary (bold); mean shown underneath in muted text.

Multi-model caveat: a session that used more than one model produces one row per model in the underlying data — so a single session with N models appears in N model columns here (once per model it touched), not just once.

Category × model usage share

Row-normalized by default: within a model, the % of its sessions in each task category (rows sum to 100%). Toggle to column-normalized: within a category, the % of sessions that ran on each model (columns sum to 100%). Cohort-size caveat applies — small cohorts make percentages noisy.

Output tokens over time

Each point is one (session, model) row. Hover for session id, project, and output tokens. Output tokens only — cache tokens are never included here. All sessions — filters apply to the comparison table only.

Tool usage

Top 10 tool names per model, from main-chain tool calls (subagent tool calls not included here). Remaining tools bucketed into "other". All sessions — filters apply to the comparison table only.

Message patterns

Soft signals from a v2 collector re-run — cadence and shape, not sentiment or quality. Hidden on v1 data.json. All sessions — filters apply to the comparison table only.
